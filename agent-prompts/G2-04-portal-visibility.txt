TASK: Build a per-project visibility control system for the client portal.

Current state:
- Client portal lives at src/app/portal/ with sub-routes: /projects/[id], /estimates/[id], /contracts/[id]
- Portal layout is at src/app/portal/layout.tsx
- Portal is excluded from auth middleware (see src/middleware.ts)
- Project settings page exists at src/app/projects/[id]/settings/

Build:
1. PRISMA MODEL — Add a PortalVisibility model to prisma/schema.prisma:
   - projectId (unique, links to Project)
   - Boolean fields: showSchedule, showFiles, showDailyLogs, showEstimates, showInvoices, showContracts, showMessages
   - All default to true except showDailyLogs (false)

2. ADMIN SETTINGS UI — In the project settings page (src/app/projects/[id]/settings/), add a "Client Portal Visibility" section:
   - Toggle switches for each visibility field
   - Saves via server action to PortalVisibility record
   - Use hui-card container with clear labels

3. PORTAL ENFORCEMENT — Update the portal project detail page:
   - Fetch PortalVisibility for the project
   - Conditionally show/hide tabs and sections based on the toggles
   - If a client navigates directly to a hidden section URL, show a friendly "This section is not available" message

4. DEFAULT BEHAVIOR — When no PortalVisibility record exists, show everything except dailyLogs (match the defaults).

Use hui-card, hui-btn for save, toggle switches with Tailwind for the admin settings.
