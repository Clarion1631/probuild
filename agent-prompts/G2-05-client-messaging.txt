TASK: Build a messaging system between clients and project managers.

Build:
1. PRISMA MODELS — Add to schema.prisma:
   - MessageThread: id, projectId (links to Project), createdAt
   - Message: id, threadId, senderType ("CLIENT" | "TEAM"), senderName, senderEmail, body, readAt, createdAt

2. API ROUTES at src/app/api/messages/:
   - GET /api/messages?projectId=X — list messages for a project thread
   - POST /api/messages — send a message (body: projectId, body, senderType)
   - PATCH /api/messages/[id]/read — mark message as read

3. ADMIN MESSAGE VIEW — Add a "Messages" tab in the project inner sidebar (src/components/ProjectInnerSidebar.tsx):
   - Page at src/app/projects/[id]/messages/page.tsx
   - Chat-style UI: messages in a scrollable container, input at bottom
   - Team messages aligned right (green bubble), client messages aligned left (gray bubble)
   - Show unread count badge on the sidebar tab

4. PORTAL MESSAGE VIEW — Add a "Messages" tab to the portal project page:
   - Similar chat UI but client messages are on the right
   - Input field at bottom to compose and send

5. EMAIL NOTIFICATION — When a new message is sent:
   - If from client → email CompanySettings.notificationEmail
   - If from team → email the Client.email
   - Use src/lib/email.ts for sending

Guard the Messages tab visibility behind PortalVisibility.showMessages (from Prompt #4). Use hui-card for the chat container.
