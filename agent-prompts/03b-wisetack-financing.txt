TASK: Integrate Wisetack consumer financing into the client portal.

CONTEXT: Wisetack is a contractor-focused financing platform. Clients apply for a loan ($500–$25,000+), get approved in seconds, and the contractor gets paid in full upfront. The client pays Wisetack back over time. The contractor pays $0 in fees — the borrower pays interest.

Wisetack provides either:
A) An embeddable widget URL (iframe) that you place in your portal
B) A partner API for creating financing requests programmatically

Build:

1. WISETACK SETUP:
   - Add env vars to .env.example: WISETACK_PARTNER_ID, WISETACK_API_KEY, WISETACK_ENVIRONMENT (sandbox/production)
   - Create src/lib/wisetack.ts with:
     * API client configured for sandbox (api.sandbox.wisetack.us) or production (api.wisetack.us)
     * Function: createFinancingRequest(amount, customerEmail, customerName, projectDescription) → returns a financing URL
     * Function: getFinancingStatus(transactionId) → returns approval status

2. PRISMA MODEL — Add FinancingRequest to schema.prisma:
   - id, invoiceId (links to Invoice), projectId
   - provider: "WISETACK" (expandable for future providers)
   - externalId (Wisetack transaction ID)
   - amount, status ("PENDING" | "APPROVED" | "FUNDED" | "DECLINED" | "EXPIRED")
   - applicationUrl (the link client clicks to apply)
   - createdAt, updatedAt

3. API ROUTES:
   - POST /api/financing/request — creates a Wisetack financing request for an invoice
     * Calls Wisetack API to create the transaction
     * Stores the FinancingRequest record with the returned application URL and transaction ID
     * Returns the application URL to the client
   - GET /api/financing/status/[id] — checks current status from Wisetack API
   - POST /api/webhook/wisetack — webhook endpoint for Wisetack status updates
     * On "funded": mark the related Invoice as "Paid", update PaymentSchedule records
     * On "declined"/"expired": update FinancingRequest status
     * Send notification email to CompanySettings.notificationEmail

4. PORTAL UI — On the client portal invoice/project page:
   - Below the "Pay Now" button (Stripe), add a "Finance This Project" section:
     * Card with Wisetack branding: "Need financing? Apply in 60 seconds."
     * "Check My Rate" button — calls /api/financing/request, then redirects to the Wisetack application URL
     * Show financing status if a request already exists (Pending, Approved, Funded, Declined)
     * If approved/funded, show green banner: "Financing approved — your contractor has been paid!"
   - Make this section toggleable from the admin payment settings (same settings page from Prompt #3)

5. ADMIN VIEW — On the invoice detail page (admin side):
   - Show financing status if a FinancingRequest exists for this invoice
   - Badge: "Client applied for financing" with current status
   - Link to Wisetack dashboard for details

6. SETTINGS — In the payment settings UI (from Prompt #3):
