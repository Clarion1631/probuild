STEP 2 — AUDIT BEFORE BUILDING:
3. BEFORE writing any code, check if this feature (or parts of it) already exists.

5. If it partially exists: extend it — do NOT rebuild from scratch.

7. Do NOT rebuild, re-scaffold, or duplicate anything that already exists.
8. ADD new server actions to src/lib/actions.ts (do NOT create new action files).
9. EXTEND existing pages/components when possible rather than creating new ones.

STEP 4 — VERIFY:
10. Do NOT run `npm run dev` or spin up a local server. Proceed directly to deploying to Vercel/Git.
11. Run `npm run build` to confirm there are no build errors.

STEP 5 — DEPLOY (MANDATORY):
12. Stage all changes: git add -A
13. Commit with a descriptive message: git commit -m "feat: <short description of what was built/fixed>"
14. Push to trigger Vercel deployment: git push origin main
15. Confirm the push succeeded and report the commit hash.
16. Wait 30 seconds then verify the Vercel deployment URL is live (check https://app.goldentouchremodeling.com or the preview URL).
16b. SELF-TESTING: You MUST use your browser_subagent tool to navigate to the live Vercel URL. Click through the feature you just built to verify it works and there are no UI bugs. If you find bugs, fix them and push again.
